Triggered CLIENT SIDE.

Event triggered when the player net data of all players has been shared on the network. This includes "Already connected players" to "New player", and "New player" to "Already connected players". It is triggered at the end of SharePlayerNetDataClientRpc(string, PlayerNetData) and SharePlayerNetDataClientRpc(string, PlayerNetData, ClientRpcParams). It is triggered when the number of player net data loaded on the client side is equal to playerNetDataCount.

playerGUIDToPlayerNetData

A map between a unique player ID and the data of the player.

private Dictionary<Guid, PlayerNetData> playerGUIDToPlayerNetData

Field Value

Dictionary<Guid, PlayerNetData>

Remarks

Client IDs are assigned whenever a client connects to the server. So, if a player leaves and reconnects, it will have a different client ID. That is why we use the unique player ID to retrieve the network data of the player. But the unique player ID is only known by the player itself. So you can use clientIDToPlayerGUIDMap(updated when a client connects to the server) to get the unique player ID of a client.
